# Постановка задачи

Необходимо разработать консольное приложение на языке Python для учёта
работы автопарка. Программа должна обеспечивать хранение и обработку
данных об **автомобилях, водителях, рейсах и техническом обслуживании**.

**Требования к программе:**

1. Должен быть реализован полностью весь функционал, описанный в задании.

2. Весь код должен быть написан в парадигме ООП и по максимуму
   реализовывать принципы **инкапсуляции, наследования и полиморфизма**.

3. Обязательно организовать хранение данных в каком-либо файловом формате.
   Формат необходимо продумать так, чтобы он соответствовал решаемой задаче.

4. Пользовательский интерфейс — консольный. Должно быть хотя бы в общих
   чертах понятно, какие пункты выбирать, чтобы получить желаемый результат.

**Применяемые принципы ООП:**

- **Инкапсуляция** — сокрытие полей классов и доступ к ним через свойства.
- **Наследование** — выделение общего базового класса.
- **Полиморфизм** — единообразные методы с разным поведением.

# Анализ задачи и структура классов

## Анализ задачи

В предметной области автопарка были выделены четыре основные сущности:

- **Водитель** — человек, управляющий автомобилем;
- **Автомобиль** — транспортное средство;
- **Рейс** — поездка конкретного водителя на конкретном автомобиле;
- **Техническое обслуживание** — ремонт или обслуживание автомобиля.

У водителя есть имя и телефон (как у любого человека), но добавляется специфическое поле —
категория прав. Это позволило применить **наследование**: создать базовый
класс `Person` и наследника `Driver`.

Остальные сущности (`Car`, `Trip`, `Maintenance`) не имеют общего предка,
так как между ними нет логической связи «является» (is-a). Наследование к ним
не применялось намеренно, чтобы не усложнять структуру искусственно.

Связи между объектами реализованы **через идентификаторы (ID)**:
- рейс хранит ID водителя и номер машины;
- запись о ТО хранит номер машины.

Для хранения данных между запусками был выбран **формат JSON**, так как он
удобен для сохранения структурированных данных (вложенных списков и словарей)
и поддерживается стандартной библиотекой Python.

## Структура классов

### 1. Класс `Person` (базовый)
Базовый класс, описывающий человека.
- **Поля:** `_name` (имя), `_phone` (телефон) — закрытые (инкапсуляция).
- **Свойства:** геттеры и сеттеры через `@property`.
- **Метод `info()`** — возвращает строку с информацией (переопределяется
  в наследниках — полиморфизм).

### 2. Класс `Driver` (наследник `Person`)
Описывает водителя.
- **Наследует** имя и телефон от `Person` (через `super().__init__`).
- **Добавляет поля:** `_driver_id` (уникальный ID), `_license_category`
  (категория прав).
- **Методы:** `info()` (переопределён), `to_dict()` / `from_dict()`
  (для сохранения в JSON и загрузки).

### 3. Класс `Car`
Описывает автомобиль.
- **Поля:** `_number` (гос. номер — используется как идентификатор),
  `_brand` (марка), `_year` (год выпуска).
- **Методы:** `info()`, `to_dict()`, `from_dict()`.

### 4. Класс `Trip`
Описывает рейс. Связывает водителя и автомобиль.
- **Поля:** `_trip_id` (ID рейса), `_driver_id` (ID водителя),
  `_car_number` (номер машины), `_route` (маршрут), `_date` (дата).
- **Методы:** `info()`, `to_dict()`, `from_dict()`.

### 5. Класс `Maintenance`
Описывает запись о техническом обслуживании.
- **Поля:** `_maint_id` (ID), `_car_number` (номер машины),
  `_description` (описание работ), `_date` (дата), `_cost` (стоимость).
- **Методы:** `info()`, `to_dict()`, `from_dict()`.

### 6. Класс `AutoPark` (главный класс-менеджер)
Управляет всей системой. Хранит списки всех объектов и реализует логику.
- **Поля:** `_drivers`, `_cars`, `_trips`, `_maintenances` — списки объектов.
- **Группы методов:**
  - **Работа с файлом:** `save()`, `load()` — сохранение и загрузка JSON.
  - **Генерация ID:** `_next_driver_id()`, `_next_trip_id()`,
    `_next_maint_id()` — выдают новый уникальный ID (на основе максимального).
  - **Добавление:** `add_driver()`, `add_car()`, `add_trip()`,
    `add_maintenance()` — с проверкой корректности ввода.
  - **Поиск:** `_find_driver()`, `_find_car()` — вспомогательные методы.
  - **Вывод:** `show_drivers()`, `show_cars()`, `show_trips()`,
    `show_maintenances()`.
  - **Удаление:** `delete_driver()`, `delete_car()`, `delete_trip()`,
    `delete_maintenance()`.
  - **Статистика:** `driver_stats()`, `car_stats()`.

### Схема связей классов
    Person  (базовый класс)
      │
      │ наследование
      ▼
    Driver ──────┐
                 │ (по driver_id)
                 ▼
               Trip ◄──────── Car (по car_number)
                               ▲
                               │ (по car_number)
                          Maintenance
